## 4. 기능 상세

### 4.1 오늘의 질문 · 이야기

- 질문 풀에서 하루 1개 제공 (**질문 리스트는 별도 정리 예정**)
- 질문만 보고 닫기 가능
- 이야기 작성 시:
  - 텍스트 — **최대 5,000자**
  - 사진 **최대 8장**, 용량 제한 없음
  - 공개: 나만 보기(기본) / 커뮤니티 공개 _(커뮤니티 공개 시 **이메일 인증 필수**)_
  - (선택) 타임캡슐: 1·3·6·12개월 또는 직접 날짜
- **같은 질문 받지 않기(F14) ON** 시 이미 받은 질문 제외. 풀 고갈 시 **90일 이상 경과한 질문 중 가장 오래된 것** 재사용 _(1년 Echo 질문은 예외)_

### 4.2 1년 Echo

- `user_question_log.shown_at` 기준 **동일 월·일 다음 해**에 같은 `question_id` 재등장  
  _(예: 2026-05-30 최초 노출 → 2027-05-30 Echo. 윤년 2/29는 2/28 fallback)_
- `annual_key`는 질문 그룹·관리용. Echo 트리거는 **shown_at + 1년(달력)** 기준
- **같은 질문 받지 않기(F14)와 무관**하게 Echo는 유지
- 1년 전 이야기 없음 → 이번이 첫 이야기

### 4.3 타임캡슐

- `unlock_at`까지 본문·사진 열람 불가
- 해제일 푸시 → P8에서 열람
- Free: **봉인 중(active) 1개**까지 _(해제된 캡슐은 한도에 포함하지 않음)_ / 광고 제거: **무제한**

### 4.4 커뮤니티

- **공개로 올린 이야기만** feed에 노출
- 질문 검색, 작성일, 작성자 닉네임 표시
- **댓글·DM·좋아요 없음** (MVP)
- **신고** → 동일 글 **3건 누적 시 feed 자동 숨김** + 관리자 검토 큐. 신고자에게는 즉시 숨김 처리

### 4.5 설정

| 설정                | 기본값       |
| ------------------- | ------------ |
| 같은 질문 받지 않기 | OFF          |
| 알림                | OFF          |
| 알림 시간           | 20:00 (로컬) |
| 글자 크기           | 중           |

**알림 UX:** 설정에서 ON → OS 권한 요청. 최초 접속 시 1회 "이야기 알림 받을까요?" (거절 시 OFF 유지). MVP에서는 알림 시간 **20:00 고정**

### 4.6 계정 · 인증

OAuth(카카오·구글 등) **사용하지 않음**. Play Store 출시에 Google 로그인 **불필요**.

| 모드                   | 설명                                                                                                           |
| ---------------------- | -------------------------------------------------------------------------------------------------------------- |
| **게스트 (기본)**      | 최초 실행 시 `device_id`(UUID) 발급. **로그인 없이** F1~F16 사용                                               |
| **회원가입**           | 이메일 + 비밀번호 (+ 닉네임). Supabase Auth Email                                                              |
| **로그인**             | 이메일 + 비밀번호                                                                                              |
| **이메일 인증**        | **소프트 verify** — 가입·로그인·private 이용은 미인증 가능. **커뮤니티 공개·IAP·비밀번호 재설정** 시 인증 필수 |
| **게스트 → 회원 병합** | 가입·로그인 시 현재 `device_id`의 이야기·설정을 회원 계정에 **이전**                                           |
| **비밀번호 재설정**    | 이메일 링크 (P16) — **인증된 계정만**                                                                          |
| **계정 삭제**          | 설정에서 요청 가능 (Play 정책)                                                                                 |
| **IAP**                | 광고 제거 구매 **전** **회원 로그인 + 이메일 인증** 필수 (영수증 ↔ user_id 연동·기기 변경 복원)                |

**Play Console 심사:** 로그인·회원 기능 제공 시 **테스트용 이메일/비밀번호** 제출 ([로그인 정보 제공 가이드](https://support.google.com/googleplay/android-developer/answer/15748846))

## 5. 사용자 흐름

### 최초 실행

```
설치 → device_id 발급(게스트) → 온보딩(P1) → (선택) 알림 → 홈(P2)
```

### 매일

```
홈(P2) → [나중에] or [이야기 쓰기(P3)] → 저장 → (Free) 광고
```

### 회원가입 · 로그인 (선택)

```
설정(P13) → 회원가입(P14) or 로그인(P15)
  → 게스트 데이터 병합 → 회원 상태
```

### 1년 Echo

```
홈(P2) → Echo 질문 → 비교(P5) → (선택) 새 이야기(P3)
```

### 커뮤니티

```
feed(P9) → 검색 → 상세(P10)
내 공개 글(P11)
```

### 광고 제거

```
설정(P13) → (미로그인) 로그인(P15) 유도
  → 로그인 + 이메일 인증 → IAP(F19, ₩4,900) → 평생 ad_free
```

## 6. 데이터 모델 (초안)

### users

```
id
device_id (nullable, unique) — 게스트 식별
email (nullable, unique) — 회원
account_type: guest | member
email_verified (boolean) — Supabase Auth 연동
nickname
ad_free (boolean, lifetime)
font_size, skip_repeat_questions (default false)
notifications_enabled (default false)
created_at, upgraded_at (nullable) — 게스트→회원 시각
```

> 비밀번호는 Supabase Auth 등 **인증 레이어**에서 관리. DB에 평문 저장 금지.

### questions

```
id, text, annual_key
```

### stories (이야기)

```
id, user_id, question_id
body_text (max 5000 chars), photo_urls (jsonb, max 8)
visibility: private | community
is_capsule, unlock_at, is_capsule_active (봉인 중 여부)
created_at
```

### user_question_log

```
user_id, question_id, shown_at — Echo 트리거 기준일
```

### story_reports (커뮤니티 moderation)

```
id, story_id, reporter_user_id, created_at
— 동일 story_id 3건 → feed 숨김
```

## 7. 기술 방향

| 항목   | 권장                                                     |
| ------ | -------------------------------------------------------- |
| 프론트 | Next.js (PWA)                                            |
| 앱     | Capacitor WebView                                        |
| 백엔드 | Supabase (Auth, Postgres, Storage)                       |
| Auth   | **Supabase Auth Email/Password** + device_id 게스트 병합 |
| IAP    | Play / App Store, 일회 구매 검증                         |
| 광고   | AdMob                                                    |
| 푸시   | FCM / APNs                                               |

### device_id + 회원 병합 (안)

1. 게스트: `device_id` → Supabase anonymous session 또는 `guests` row
2. 회원가입/로그인: email auth 성공 → `device_id` 소유 stories·settings를 `user_id`로 migrate
3. IAP: `ad_free`를 **회원 user_id**에 저장, 영수증 token 보관

## 8. 운영 · 정책

- **private 이야기**: 본인만 열람
- **community 이야기**: 전체 열람, 신고 가능 (3건 누적 시 자동 숨김)
- 사진: **최대 8장**, 용량 제한 없음 (신고·moderation으로 대응)
- **계정 삭제** 요청 지원 (Play·개인정보)
- **개인정보처리방침**: email, device_id 수집·이용 명시
- 스토어 청소년 등급 준수
