## 공통 정책

### 시간·질문

- 모든 「오늘」「하루」 기준은 **KST(한국 표준시)** 입니다.
- 질문 풀은 **365개**이며, `month`·`day` 조합으로 당일 질문을 조회합니다.
- 같은 KST 날짜에는 **질문 하나**만 배정됩니다.

### 계정·접근 권한

| 동작                               | 게스트 | 회원 (미인증) | 회원 (이메일 인증) |
| ---------------------------------- | ------ | ------------- | ------------------ |
| 오늘의 질문 보기                   | ✅     | ✅            | ✅                 |
| private 이야기 작성·서랍·타임캡슐  | ✅     | ✅            | ✅                 |
| Story `visibility=community` 저장  | ❌     | ❌            | ✅                 |
| CommunityPost 작성·수정            | ❌     | ❌            | ✅                 |
| StoryComment·CommunityComment 작성 | ❌     | ❌            | ✅                 |
| 닉네임 변경·설정                   | ✅     | ✅            | ✅                 |

- **게스트:** `device_id`(localStorage) → `POST /users/guest`로 자동 생성. 로그인 없이 대부분 기능 이용.
- **회원 전환:** signup/login 시 게스트 데이터 병합(merge).
- **회원탈퇴:** 설정에서 계정 삭제 가능.

### 콘텐츠 공통 제한

- 본문: **최대 5,000자**
- 사진: **최대 8장**, S3 presigned URL 업로드
- 신고: **동일 글 3건** 누적 시 `hiddenFromFeed=true` (공개 feed·목록에서 숨김, Story·CommunityPost 각각)
- DM(1:1 메시지): **없음**

### 커뮤니티 이원화 (핵심 결정)

| 구분           | 데이터                             | Feed         | 상세 Route        | 댓글             | SEO     |
| -------------- | ---------------------------------- | ------------ | ----------------- | ---------------- | ------- |
| **공개 Story** | `stories` (`visibility=community`) | 홈 `/`       | `/public/[id]`    | StoryComment     | index   |
| **토론**       | `community_posts`                  | `/community` | `/community/[id]` | CommunityComment | noindex |

> `/app/*` Route는 `/`로 redirect됩니다. 아래 Route는 canonical 경로 기준입니다.

## 홈 `/`

Shell 탭 「오늘의 질문」(중앙). AppHeader, 공개 Story feed, FAB(이야기 쓰기), 하단 탭·광고 띠(placeholder).

### 오늘의 질문

- **기능:** KST 기준 오늘 질문 1개 표시, 「이야기 쓰기」 진입
- **정책**
  - 답을 쓰지 않아도 됨 (질문 노출만으로 OK)
  - DB 미연결 시 fallback 문구 표시
- **미구현:** 온보딩(P1), F14 같은 질문 받지 않기

### 공개 Story feed

- **기능:** `visibility=community` Story 전역 목록, 카드 클릭 → `/public/[id]`
- **정책**
  - `hiddenFromFeed=true` 글은 목록 제외
  - 본인·타인 구분 없이 시간순 feed
  - 검색 UI 없음 (토론 탭만 검색)
- **미구현:** 「내가 공개한 이야기」 필터(P11 deprecated)

### FAB · 광고

- FAB: `/write` 이동
- 광고: shell 레이아웃 하단 50px placeholder (AdSense 미연동)

## 이야기 작성 `/write` · `/write/[id]`

전체 화면 작성 레이아웃. 오늘의 질문·공개 범위·사진·타임캡슐 전환.

### 새 이야기 작성 `/write`

- **기능:** 텍스트·사진·공개 범위(private/community)·타임캡슐 다이얼로그
- **정책**
  - **오늘의 질문에만** 일반 Story 생성 가능 (`QUESTION_NOT_TODAY` 409)
  - 같은 KST 날, 같은 질문에 **이야기 1개만** (`TODAY_STORY_EXISTS` 409)
  - 기본 공개 범위: **private(나만 보기)**
  - `visibility=community` 저장: **이메일 인증 필수**
  - 타임캡슐은 별도 Route(`/capsule/write`)에서 `isCapsule=true`로 생성
- **제한:** 본문 5,000자, 사진 8장

### 이야기 수정 `/write/[id]`

- **기능:** 기존 private Story 수정
- **정책**
  - **본인 Story만** 수정 가능
  - **private Story만** PATCH 지원 (공개 Story 수정 UI 없음)
  - 질문·날짜 변경 불가 — 당일 작성분만 수정 대상
- **미구현:** 공개 Story 수정·삭제 전용 UI

## 서랍 `/drawer` · `/drawer/[id]`

Shell 탭 「서랍」. private Story 모음.

### 서랍 목록 `/drawer`

- **기능:** 월별 섹션, 검색, 북마크 필터, 잔디(활동), 날짜 선택, 회상 카드(DrawerRevisitCard)
- **정책**
  - `visibility=private` Story만 표시 (타임캡슐 포함)
  - 본인 데이터만 조회
  - Echo Story는 `isEchoStory` 뱃지 표시

### Story 상세 `/drawer/[id]`

- **기능:** 본문·사진·메타·북마크·글자 크기·삭제
- **정책**
  - **본인만** 열람·수정·삭제
  - 타임캡슐이 봉인 중이면 서랍 목록에는 보이나 상세에서 본문 숨김(캡슐 상세로 이동)
- **미구현:** Echo 1년 비교 UI(P5)

### Echo

- **정책:** 동일 `question_id`를 2회 이상 받으면 `isEchoStory=true` (뱃지만, 비교 화면 없음)
- **미구현:** P5 비교 UI, Echo 알림 Cron

## 타임캡슐 `/capsule` · `/capsule/write` · `/capsule/[id]`

Shell 탭 「타임캡슐」+ detail 작성·상세.

### 목록 `/capsule`

- **기능:** 봉인·해제 캡슐 카드, FAB → 작성
- **정책**
  - 본인 캡슐만 표시
  - **개수 제한 없음**
  - 서랍 배너에서도 진입 가능

### 작성 `/capsule/write`

- **기능:** 수신자(선택)·봉인 기간·본문·사진·봉인 확정
- **정책**
  - `questionId: null`, `isCapsule: true`, `unlock_at` 필수
  - `visibility`는 항상 **private**
  - 오늘의 질문·1일 1Story 제한 **적용 안 함**
  - **이메일 인증 불필요**

### 상세 `/capsule/[id]`

- **기능:** 봉인 오버레이 / 해제 후 본문·삭제
- **정책**
  - `unlock_at` 이전: 본문·사진 **숨김**
  - `unlock_at` 이후: 일반 Story처럼 열람
  - 본인만 삭제 가능
- **미구현:** 해제일 푸시 알림

## 커뮤니티 토론 `/community` · `/community/[id]` · `/community/write`

Shell 탭 「커뮤니티」. CommunityPost 전용 (공개 Story와 별도).

### 토론 feed `/community`

- **기능:** 글 목록, 검색(닉네임·본문), FAB → 작성
- **정책**
  - `hiddenFromFeed=false` 글만 표시
  - 시드: 관리자 **환영·안내 글 1건** (`seedWelcomeCommunityPost`)
  - 재시드 시 환영 글 ID 외 CommunityPost 삭제

### 토론 상세 `/community/[id]`

- **기능:** 본문(줄바꿈 유지), 사진, 5종 반응, 댓글·답글·멘션, 신고, 수정·삭제(본인)
- **정책**
  - **댓글 답글 깊이 1단계** (`REPLY_DEPTH_EXCEEDED` — 답글에 답글 불가)
  - 멘션: `@닉네임` (1~20자, 한글·영문·숫자)
  - 반응 5종: heart, sad, angry, fire, clap
  - 신고 3건 → feed 숨김
  - **이메일 인증** 필요: 글·댓글 작성
- **미구현:** Story 댓글 이모지 UI (API만 존재)

### 토론 작성·수정 `/community/write` · `/community/write/[id]`

- **기능:** 본문·사진·질문 연결(선택)·이메일 인증 다이얼로그
- **정책**
  - 작성·수정: **본인 + emailVerified**
  - `questionId` 선택 optional
  - 본문 5,000자, 사진 8장

## 공개 Story 상세 `/public/[id]`

SEO index 대상. `visibility=community` Story.

### 기능·정책

- **기능:** 본문·사진·질문·댓글·답글·5종 반응·신고
- **정책**
  - 누구나 열람 (로그인 불필요)
  - 댓글 작성: **이메일 인증** 필요
  - 답글 깊이 1단계
  - 신고 3건 → feed 숨김
  - robots: **index, follow**
- **미구현:** 공개 Story 수정·삭제 UI, Story 댓글 반응 UI

## 설정 `/settings` · `/settings/login` · `/settings/signup`

Shell 탭 「설정」.

### 설정 메인 `/settings`

| 기능          | 정책                                                            |
| ------------- | --------------------------------------------------------------- |
| 프로필·닉네임 | 게스트: 자동 닉네임+숫자. 회원: 변경 가능, unique               |

- **미구현:** F14 같은 질문 받지 않기, F19 광고 제거 IAP, 비밀번호 재설정(P16)

### 로그인·회원가입

- **기능:** Email/Password (Supabase Auth)
- **정책:** signup/login 성공 시 **게스트 → 회원 merge**
- **미구현:** 비밀번호 재설정

## 소개·질문 아카이브 `/about` · `/questions`

Marketing 레이아웃 (탭·광고 없음).

### 서비스 소개 `/about`

- **기능:** 서비스 설명, Echo·규칙·FAQ, 정책 링크

### 질문 모아보기 `/questions` · `/questions/[questionId]`

- **기능:** 365 질문 목록, 날짜별 질문·해당 공개 Story feed
- **정책**
  - 질문 아카이브 열람 (오늘 질문으로 redirect 아님)
  - SEO index
- **주의:** 아카이브에서 보는 Story는 **열람만** — 작성은 오늘 질문(`/write`)에서만
