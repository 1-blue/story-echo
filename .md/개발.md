## 1. 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────┐
│                        Clients                              │
│  ┌──────────────────┐       ┌──────────────────┐           │
│  │  apps/web        │       │  apps/mobile     │           │
│  │  Next.js (PWA)   │       │  Expo (RN) ⏳    │           │
│  │  SEO · AEO       │       │  IAP · Push · Ad │           │
│  └────────┬─────────┘       └────────┬─────────┘           │
│           │                          │                      │
│           └──────────┬───────────────┘                      │
│                      ▼                                      │
│           packages/api-client (orval 생성)                  │
│           packages/schemas   (zod — 공유 검증)              │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  apps/web — Next.js Route Handlers (BFF / API)              │
│  zod 검증 · Prisma · Supabase Auth 토큰 검증                │
└──────────────┬──────────────────────────────┬───────────────┘
               ▼                              ▼
     Supabase Postgres              AWS S3 (이미지)
     (Prisma ORM)                   presigned URL 업로드
               │
     Supabase Auth (Email/Password)
```

**핵심 원칙**

- **웹·앱 UI는 분리**, **API·타입·스키마는 공유** (monorepo)
- DB 접근은 **서버(Route Handler)에서만**
- 인증: **Supabase Auth**, 데이터: **Prisma**
- API 클라이언트: **orval** (타입 + TanStack Query hooks)
- 웹 UI: **shadcn/ui** + BottomSheet + Motion/Magic UI + [DESIGN.md](./DESIGN.md) 토큰
- SSOT: zod → openapi.json → orval ([§2.7](#27-개발-원칙--ssot-single-source-of-truth))

### 6.2 Detail

| Route | 주요 mutate invalidate |
| --- | --- |
| `/app/drawer/[id]` | bookmark PATCH story |
| `/app/public/[id]` | comments, reactions → public id + public list |
| `/app/community/[id]` | comments, reactions, delete → post id + posts list |
| `/app/capsule/[id]` | delete → capsule + drawer |
| `/app/notifications` | read → notifications |

### 6.3 Write

| Route | gate | mutate |
| --- | --- | --- |
| `/app/write` | community → emailVerified | POST stories → drawer, public |
| `/app/write/[id]` | 동일 | PATCH stories |
| `/app/capsule/write` | — | POST capsule → drawer |
| `/app/community/write` | emailVerified | POST posts |
| `/app/community/write/[id]` | emailVerified | PATCH posts |

### 6.4 Auth

| Route | 테스트 |
| --- | --- |
| `/app/settings/login` | login, merge, error |
| `/app/settings/signup` | signup, merge, validation |

### 6.5 연관 그룹 mutate 규칙

| 액션 | invalidate 대상 |
| --- | --- |
| Story create (community) | drawer, public list |
| Story comment | public detail, public list (commentCount) |
| Community comment | post detail, posts list |
| Community post delete | posts list, router back |
| Capsule create/delete | drawer, capsule list |
| Nickname/font/notify | users/me |

### 6.6 권한 · gate 테스트

| 조건 | 기대 |
| --- | --- |
| Guest | drawer, write(private), read public/community |

| Non-owner story | GET ok (public), PATCH 403 |
| Report 3x same target | hiddenFromFeed, feed에서 제외 |

## 8. 오늘의 질문 · Echo

### 현재 구현 (`lib/today-question.ts`)

```ts
// KST (month, day) → prisma.question.findUnique({ month_day })
// 윤년 2/29 → 2/28과 동일 질문 (normalizeMonthDay)
// DB 비어 있으면 FALLBACK_QUESTION
```

- 시드 ID: `@storyecho/database/question-calendar` — day-of-year → UUID
- 질문 데이터: `packages/database/src/seeds/questions.data/` (365건)

### Echo (`lib/story-mapper.ts`)

- `fetchEchoCountsForUser` — 동일 questionId 2회 이상 작성
- `isEchoStoryFromCounts` → drawer/detail DTO `isEchoStory`
- **P5 비교 UI 없음**

### API integration 테스트 (Vitest)

- **방식:** `next build` → `next start` → `fetch(/api/v1/...)` + **실제 Supabase Postgres** (업계에서 흔한 full-stack integration)
- **위치:** `apps/web/tests/integration/api/*.test.ts`
- **실행:** `pnpm --filter web build` 후 `pnpm --filter web test:integration`
- **global-setup:** `.next/BUILD_ID`가 있으면 **build 생략**(CI 이중 빌드 방지), 없으면 로컬에서 1회 build 후 start. 이미 떠 있는 서버는 health 체크 후 재사용
- **env:** `packages/database/.env` + Supabase secrets ([`.github/workflows/README.md`](../.github/workflows/README.md))
- **인증:** 게스트는 `X-Device-Id`만 — 세션 쿠키 없을 때 Supabase `getUser()` 생략(불필요한 Auth RTT 제거). 로그인 E2E/통합은 쿠키 경로 유지
- **검증:** Zod `parse-api` 헬퍼 (`@storyecho/schemas` SSOT)
- **Arrange:** 게스트·시드 데이터는 `ensureGuestInDb` 등 Prisma 헬퍼 가능 — **Act/Assert는 HTTP** (`apiFetch`). 무거운 케이스(예: 3회 report)는 `try/finally`로 `cleanupTestUserByDeviceId` 보장
- **타임아웃:** `testTimeout` 90s, 무거운 단일 케이스는 `it(..., 120_000)`. `TEST_SLOW_MS`(기본 10s) 초과 시 `apiFetch` warn
- **직렬:** `fileParallelism: false` — 같은 dev DB를 쓰므로 데이터 격리 우선
- **느린 이유(정상):** 케이스마다 HTTP·DB 여러 번 + CI 러너↔DB 지리적 거리. 방식 오류(이중 build, 매 요청 getUser)는 위 설정으로 제거

### E2E · 로컬

```bash
pnpm --filter web build && pnpm --filter web test:integration
pnpm --filter web exec playwright install chromium && pnpm --filter web test:e2e
```
